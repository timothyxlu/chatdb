Claude 3.5 2:34 PM
Great project! Here's an architecture I'd recommend for a Next.js + Cloudflare D1 chat history app with GitHub auth:

Tech Stack:
• Next.js 15 with App Router
• NextAuth.js v5 for GitHub OAuth
• Cloudflare D1 (SQLite-compatible) for production
• better-sqlite3 for local development
• Drizzle ORM for type-safe queries across both
typescript — schema.ts
export const users = sqliteTable('users', {
  id: text('id').primaryKey(),
  githubId: text('github_id').unique(),
  username: text('username').notNull(),
  avatarUrl: text('avatar_url'),
  createdAt: integer('created_at'),
});

// Chat sessions table
export const sessions = sqliteTable('sessions', {
  id: text('id').primaryKey(),
  userId: text('user_id').references(() => users.id),
  agentName: text('agent_name').notNull(),
  title: text('title'),
  updatedAt: integer('updated_at'),
});

How do I handle the database connection differently for local development vs Cloudflare production?
🟠
Claude 3.5 2:42 PM
The cleanest approach is to abstract the database behind a single getDb() function that returns the right driver based on the environment. Drizzle ORM makes this easy since it works with both SQLite drivers.
typescript — lib/db.ts
import { drizzle } from 'drizzle-orm/better-sqlite3';
import { drizzle as d1 } from 'drizzle-orm/d1';

export function getDb(env?: CloudflareEnv) {
  if (process.env.NODE_ENV === 'production') {
    return d1(env!.DB);
  }
  // Local: use SQLite file
  const sqlite = new Database('local.db');
  return drizzle(sqlite);
}
